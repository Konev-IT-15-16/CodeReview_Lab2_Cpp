# Общие изменения для всех задач

Разделение на модули: Код разбит на три файла: Header.h (объявления), Source.cpp (реализация логики) и Main.cpp (точка входа).

Инкапсуляция: Логика структур данных (стек, очередь, списки) полностью перенесена внутрь классов. Прямое манипулирование указателями из main исключено.

Безопасность типов: Тип int заменен на int32 для обеспечения кроссплатформенной переносимости.

Дружественный интерфейс: Вместо линейного ввода реализовано интерактивное меню (switch-case в цикле while), позволяющее многократно тестировать функции без перезапуска.

Безопасный ввод: Добавлены функции проверки (IsNumber, SafeReadInt), предотвращающие зависание программы при вводе букв вместо цифр.

Стиль именования:

Классам и структурам добавлены префиксы F (например, FStack, FListNode).

Методы переименованы в PascalCase (например, AddElement вместо add).

Входящие параметры получили префикс In (например, int32 InValue).

# Подробный разбор изменений по задачам

## Task 01: Стек (FStack)

Убрано: Глобальное пространство имен using namespace std;.

Добавлено: Класс FStack с автоматическим управлением памятью в деструкторе.

Изменено: Метод topdel разделен на логическое извлечение значения и безопасное удаление узла с проверкой на nullptr.

## Task 02: Очередь (FQueue)
Убрано: Передача структуры PQueue по ссылке в каждую функцию.

Добавлено: Обработка критической ошибки извлечения из пустой очереди (теперь метод Dequeue возвращает bool, предотвращая краш).

Изменено: Упрощена логика обновления указателей Head и Tail при добавлении первого элемента.

## Task 03: Односвязный список (FLinkedList)
Убрано: Дружественные функции (friend), нарушающие инкапсуляцию. Логика вставки теперь является методом класса.

Добавлено: Полноценный деструктор для очистки всех узлов списка при выходе из программы.

Изменено: Исправлен алгоритм «вставки перед каждым вторым элементом». Теперь он корректно работает через итерацию с двумя указателями (Current и Previous), не теряя связи в списке.

## Task 04: Двусвязный список (FDoublyLinkedList)
Убрано: Названия на транслите (Uzel, Spisok, Initsializatsiya).

Добавлено: Метод удаления элемента по значению (стандартный набор методов).

Изменено: Логика поиска элементов между Min и Max вынесена в приватные вспомогательные методы. Исправлена проблема неопределенного поведения, когда Max находится в списке раньше, чем Min.

## Task 05: STL Контейнеры (std::deque, std::list)
Убрано: Цикл for с многократным вызовом метода .insert().

Добавлено: Реализовано три режима заполнения данных: с клавиатуры, генерация случайных чисел и чтение из текстовых файлов (.txt).

Изменено: Выполнено требование «один вызов insert». Для вставки последних 5 элементов дека в обратном порядке использованы обратные итераторы (rbegin, rend) и вставка диапазона (range-based insert).

Безопасность: Добавлены проверки size() >= 5 для предотвращения выхода итераторов за пределы допустимых значений.
